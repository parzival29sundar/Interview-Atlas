# 📂 Interview Atlas - Project Structure

>

> Every new feature must follow this architecture.

# Rules

## Components

One component = One file.

Never create huge files.

---

## Pages

Pages should only compose components.

Business logic belongs inside Features.

---

## Services

Every backend request belongs inside Services.

Never call Supabase directly from components.

---

## Types

Every important object must have its own interface/type.

---

## Explorer

The Explorer is the heart of the application.

Structure:

